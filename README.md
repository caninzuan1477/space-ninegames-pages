# space-ninegames-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态资源的形式存放，不依赖于特定域名或后端服务。

## 项目简介

这是一个通用型的静态页面仓库，主要用途包括：

- 存放独立的 HTML 页面及其相关资源（CSS、JavaScript、图片等）
- 提供页面归档与版本管理
- 便于通过 GitHub Pages 或其他静态托管服务直接访问

仓库本身不限定页面主题或用途，仅作为页面文件的存储与发布载体。

## 目录结构

```
.
├── index.html          # 默认入口页面
├── pages/              # 归档页面存放目录
│   ├── example/        # 示例页面文件夹
│   │   ├── index.html
│   │   └── assets/
│   └── ...
├── assets/             # 公共资源目录（可选）
└── README.md           # 本文件
```

- `index.html`：仓库根目录下的默认页面，可作为导航或说明页。
- `pages/`：存放所有归档的独立页面，每个页面可拥有独立的子目录和资源文件。
- `assets/`：存放多个页面共享的公共资源（如图片、样式、脚本等），非必需。

## 页面归档说明

- 每个归档页面应放置于 `pages/` 下独立的文件夹中，避免与其他页面文件混淆。
- 页面内引用资源时，建议使用相对路径，以确保在本地或托管环境中均可正常访问。
- 归档页面不包含任何动态服务器端逻辑，仅依赖客户端技术（HTML、CSS、JavaScript）。

## 维护说明

- 新增页面：在 `pages/` 下创建新文件夹，放入页面文件及所需资源，并更新 `index.html` 中的导航链接（如有）。
- 更新页面：直接修改对应文件夹内的文件，提交至仓库即可。
- 删除页面：移除对应文件夹，并清理相关链接引用。

如需调整目录结构或添加公共资源，请遵循现有命名与组织方式，保持仓库整洁。
